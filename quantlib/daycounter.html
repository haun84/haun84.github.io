QuantLib: DayCounter Class ​

Language

The Day Count Convention is critical for the valuation of financial products, especially for fixed-income products. QuantLib provides the following common rules

DayCounter Class Overview ​

DayCounter Class

  • In financial transactions, there are rules that all market participants have agreed on in common, and in the FICC(Fixed Income, Currency, and Commodity) market, there are a wide variety of market conventions.

  • The most important part of pricing FICC products is how to properly reflect the details of these various market conventions in implementation.

  • As with holidays, the date calculation method also varies from country to country and market, and QuantLib provides various date calculation methods through a class called DayCounter.

Day Count Convention

  • The date calculation method is a mutual promise on how to calculate interest when the interest calculation section is less than one year. This is the most important detail in the swap and bond markets, where interest is exchanged every three or six months.

  • Since the unit of notice of interest rate is the annual rate, it is necessary to basically set the time frame in annual units in order to calculate interest for less than a year.

  • In short, we can consider two ways to calculate the date, one is how many days the year will be calculated, and the other is how many days the month will be calculated.

  • First of all, there are ACT, 365, 360 and 252 as a way of calculating a year.

    • Here, ACT stands for Actual and is a method of calculating the actual number of days of the year.
    • If ACT is used, one year will be 366 days in the case of leap year.
    • The rest of the conventions are fixedly used as 365 days, 360 days, and 252 days.

  • There are ACT, 365, 360 and 252 as a way of calculating a month.

    • ACT is a method of calculating the number of days of the actual interest section, and 30 is a method of calculating a month as 30 days.
    • Finally, Business Days are a method of calculating only business days excluding holidays.
    • For example, if the 30 method is used for the section from August 2 to October 2, it will be calculated as 30 X 2 + 1 = 61 days because it is a period of 2 months and 1 day.

  • When the two elements are finally combined, they can be expressed as ACT/365, ACT/360, ACT/ACT and 30/360, which are market conventions for calculating the number of days of interest.

    • One-month convention is located in the numerator and one-year convention is located in the denominator.
    • Different countries use different conventions, with the US using the ACT/360 method, Europe using the 30/360 method, and Korea using the ACT/365 method.

DayCounter Class Structure ​

The way to implement the DayCounter class is very simple. Like the Calendar class, you just need to declare the method you want to use.
The code below is an example of each date calculation method.

python
import QuantLib as ql

# Construction
act360 = ql.Actual360()                          # ACT/360
act365 = ql.Actual365Fixed()                     # ACT/365
actact = ql.ActualActual(ql.ActualActual.ISDA)   # ACT/ACT (ISDA)
thirty360 = ql.Thirty360(ql.Thirty360.European)  # 30/360 (EU)
b252 = ql.Business252()                          # BusinessDay/252
simpledc = ql.SimpleDayCounter()                   # NASD

Day Count Convention

  • Actual360 : Actual / 360

  • Actual365Fixed : Actual / 365(Fixed)

    • NoLeap

  • ActualActual : Actual / Actual

  • Business252 : Business / 252

DayCounter Class Functions ​

The DayCounter class has two main functions.
One is the dayCount() method that calculates the date between two dates,
The other is the yearFraction() method, which calculates the period between two dates in annual units based on market convention.

dayCount() ​

The dayCount() method provides the ability to calculate the period between two dates in days based on a given date calculation method.

python
import QuantLib as ql

# Construction
act360 = ql.Actual360()                          # ACT/360
act365 = ql.Actual365Fixed()                     # ACT/365
actact = ql.ActualActual(ql.ActualActual.ISDA)   # ACT/ACT (ISDA)
thirty360 = ql.Thirty360(ql.Thirty360.European)  # 30/360 (EU)
b252 = ql.Business252()                          # BusinessDay/252
simpledc = ql.SimpleDayCounter()                   # NASD

# Function
startDate = ql.Date(15, 5, 2024)
endDate = ql.Date(15, 8, 2024)

# dayCount(Date1, Date2)
print(f'Day Count by Actual/360 = {act360.dayCount(startDate, endDate)}')
print(f'Day Count by Actual/365 = {act365.dayCount(startDate, endDate)}')
print(f'Day Count by Actual/Actual = {actact.dayCount(startDate, endDate)}')
print(f'Day Count by 30/360 = {thirty360.dayCount(startDate, endDate)}')
print(f'Day Count by BusinessDay/252 = {b252.dayCount(startDate, endDate)}')
print(f'Day Count by SimpleDayCount = {simpledc.dayCount(startDate, endDate)}')

## Output
## Day Count by Actual/360 = 92
## Day Count by Actual/365 = 92
## Day Count by Actual/Actual = 92
## Day Count by 30/360 = 90
## Day Count by BusinessDay/252 = 65
## Day Count by SimpleDayCount = 90

yearFraction() ​

The yearFraction() method provides the ability to convert the period between two given dates in units of years when a specific date calculation method is used.
Each method outputs a different result value depending on how many days a year is calculated.
In other words, the amount of interest calculated is different depending on how the market conventions method is used.

python
import QuantLib as ql

# Construction
act360 = ql.Actual360()                          # ACT/360
act365 = ql.Actual365Fixed()                     # ACT/365
actact = ql.ActualActual(ql.ActualActual.ISDA)   # ACT/ACT (ISDA)
thirty360 = ql.Thirty360(ql.Thirty360.European)  # 30/360 (EU)
b252 = ql.Business252()                          # BusinessDay/252
simpledc = ql.SimpleDayCounter()                   # NASD

# Function
startDate = ql.Date(15, 5, 2024)
endDate = ql.Date(15, 8, 2024)

# yearFraction(Date1, Date2)
print(f'Year Fraction by Actual/360 = {round(act360.yearFraction(startDate, endDate), 4)}')
print(f'Year Fraction by Actual/365 = {round(act365.yearFraction(startDate, endDate), 4)}')
print(f'Year Fraction by Actual/Actual = {round(actact.yearFraction(startDate, endDate), 4)}')
print(f'Year Fraction by 30/360 = {round(thirty360.yearFraction(startDate, endDate), 4)}')
print(f'Year Fraction by BusinessDay/252 = {round(b252.yearFraction(startDate, endDate), 4)}')
print(f'Year Fraction by SimpleDayCount = {round(simpledc.yearFraction(startDate, endDate), 4)}')

## Output
## Year Fraction by Actual/360 = 0.2556
## Year Fraction by Actual/365 = 0.2521
## Year Fraction by Actual/Actual = 0.2514
## Year Fraction by 30/360 = 0.25
## Year Fraction by BusinessDay/252 = 0.2579
## Year Fraction by SimpleDayCount = 0.25
