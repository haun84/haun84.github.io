QuantLib: InterestRate Class ​

QuantLib InterestRate class calculates the interest rate, discount factor, and compounding factor for a specific period based on the given market conventions

Language

  • 한국어 페이지: Korean Page

InterestRate Class Overview ​

Interest Rate

  • FICC refers to Fixed Income, Currency, and Commodity.
  • Here, Fixed Income refers to assets related to interest rates.
  • Interest rate is the cost or return for borrowing or lending money over a certain period.
  • However, unlike other underlying assets like stocks, exchange rates, and commodity prices, interest rates have a different characteristic.
  • Interest rates are based on a period of time, not a specific point in time

InterestRate Class

  • Interest rates must be applied to a specified period, not a specific point in time.
  • Due to this nature, discussions about holiday treatment, interest day-count conventions, and interest compounding methods are necessary.
  • In QuantLib, due to the unique nature of interest rates, a separate InterestRate class has been created specifically for interest rates.

InterestRate Class Structure ​

python
import QuantLib as ql

# ql.InterestRate(rate: Real, 
#                 dayCount: ql.DayCounter,
#                 compounding: ql.Compounding,
#                 frequency: ql.Frequency)
rate = ql.InterestRate(0.05, ql.Actual360(), ql.Compounded, ql.Annual)

InterestRate Class

  • The InterestRate class calculates the interest rate, discount factor, and compounding factor for a specific period based on the given market conventions.
  • Four arguments are required to define this class:
  • Interest rate (rate), Day-count convention (DayCounter), Compounding method (Compounding), and Payment frequency (Frequency).

Compounding

  • Compounding is an enumeration consisting of members that decide how interest compounding is applied.
  • This enumeration contains five members:
  • In the financial world, interest is generally calculated using a standard compounding method.
  • However, when modeling interest rate processes in financial engineering, the rate is often assumed to follow continuous compounding

Compounding

  • ql.Simple: Simple interest [c = 1 + rt]
  • ql.Compounded: Compounded interest [c = (1 + r/f)^(tf)]
  • ql.Continuous: Continuous compounding [c = e^(rt)]
  • ql.SimpleThenCompounded: Simple interest until the next coupon date, then compounded interest
  • ql.CompoundedThenSimple: Compounded interest until the next coupon date, then simple interest
    • c: Compounding factor
    • r: Interest rate
    • t: Time to maturity
    • f: Payment frequency per year

Korean Bond Market Compounding Convention

  • The method used to calculate the price of actual Korean bonds is similar to the fourth method, ql.SimpleThenCompounded.
  • However, the bond pricing convention in Korea differs slightly from the ql.SimpleThenCompounded method used by QuantLib.
  • This is because actual day counts are factored in when calculating the compounding factor until the next coupon date. You can check the bond pricing convention in Korea through the Korea Financial Investment Association > Regulations on the Operations and Business of Financial Investment Companies > [Annex 14: Price Conversion Formula for Bond Yields].

InterestRate Class Functions ​

Calculating Discount Factor and Compounding Factor ​

  • discountFactor(d1, d2) : Calculates the discount factor over a specified period.

  • compoundFactor(d1, d2) : Calculates the compounding factor over a specified period.

python
import QuantLib as ql

rate = ql.InterestRate(0.05, ql.Actual360(), ql.Compounded, ql.Annual)

# Discount Factor & Compound Factor
print("Rate: ", rate.rate())
print("DayCount: ", rate.dayCounter())
print("DiscountFactor: ", rate.discountFactor(1))
print("DiscountFactor: ", rate.discountFactor(ql.Date(15,6,2020), ql.Date(15,6,2021)))
print("CompoundFactor: ", rate.compoundFactor(ql.Date(15,6,2020), ql.Date(15,6,2021)))

# Output
# Rate:  0.05
# DayCount:  Actual/360 day counter
# DiscountFactor:  0.9523809523809523
# DiscountFactor:  0.9517357984551467
# CompoundFactor:  1.050711764360651

Calculating Interest Rates Under Different Market Conventions ​

  • equivalentRate(resultDC, comp, freq, d1, d2) : Returns the equivalent rate under different conditions.

    • Interest day-count convention (resultDC)
    • compounding method (comp)
    • payment frequency (freq)
    • start date (d1)
    • end date (d2)

  • impliedRate(compound, resultDC, comp, freq, d1, d2) : Given a compounding factor, calculates and returns the interest rate implied within that factor.

    • Compounding factor (compound)
    • interest day-count convention (resultDC)
    • compounding method (comp)
    • payment frequency (freq)
    • start date (d1)
    • end date (d2)
python
import QuantLib as ql

rate = ql.InterestRate(0.05, ql.Actual360(), ql.Compounded, ql.Annual)

print("Rate: ", rate.rate())
print("DayCount: ", rate.dayCounter())

print("EquivalentRate: ", rate.equivalentRate(ql.Actual360(), ql.Compounded, ql.Semiannual, ql.Date(15,6,2020), ql.Date(15,6,2021)))

factor = rate.compoundFactor(ql.Date(15,6,2020), ql.Date(15,6,2021))
print("ImpliedRate: ", rate.impliedRate(factor, ql.Actual360(), ql.Continuous, ql.Annual, ql.Date(15,6,2020), ql.Date(15,6,2021)))

# Output
# Rate:  0.05
# DayCount:  Actual/360 day counter
# EquivalentRate:  4.939015 % Actual/360 Semiannual compounding
# ImpliedRate:  4.879016 % Actual/360 continuous compounding

Reflecting Korean Market Practice: InterestRate Class ​

Korean Bond Market Compounding Convention

  • The method used to calculate the price of actual Korean bonds is similar to the fourth method, ql.SimpleThenCompounded.
  • However, the bond pricing convention in Korea differs slightly from the ql.SimpleThenCompounded method used by QuantLib.
  • This is because actual day counts are factored in when calculating the compounding factor until the next coupon date. You can check the bond pricing convention in Korea through the Korea Financial Investment Association > Regulations on the Operations and Business of Financial Investment Companies > [Annex 14: Price Conversion Formula for Bond Yields].

Below is the InterestRate class modified to reflect the Korean bond market convention

python
class InterestRate(ql.InterestRate):
    """Modify QuantLib InterestRate Class
       SimpleThenCompounded in QuantLib is not Korea Treasury Compounding Convention
       if compounding == ql.SimpleThenCompounded:
           for t <= 1/f : c = 1 + r*t
           for t > 1/f  : c = (1 + r/f)^(tf)
           where t = days / first_period_days
    """
    def __init__(self, r, dc, comp, freq):
        super().__init__(r, dc, comp, freq)
        self._r = r
        self._dc = dc
        self._comp = comp
        self._freq = freq.real
    
    def discountFactor(self, d1, d2):
        r = self._r
        dc = self._dc
        comp = self._comp
        freq = self._freq
        t = dc.yearFraction(d1, d2)
        if comp == ql.SimpleThenCompounded:
            n = int(np.divmod(t, 1/freq)[0])
            nextDate = d2 - ql.Period(int(n*12/freq), ql.Months)
            prevDate = d2 - ql.Period(int(n*12/freq + 12/freq), ql.Months)
            if (t <= 1.0/freq):
                df = 1 / (1.0 + r/freq * (nextDate-d1) / (nextDate-prevDate))
            else:
                df = 1 / ((1.0 + r/freq * (nextDate-d1) / (nextDate-prevDate)) * ((1 + r/freq)**n))
        else:
            df = super().discountFactor(d1, d2)
        return df
    
    def compoundFactor(self, d1, d2):
        r = self._r
        dc = self._dc
        comp = self._comp
        freq = self._freq
        t = dc.yearFraction(d1, d2)
        if comp == ql.SimpleThenCompounded:
            n = int(np.divmod(t, 1/freq)[0])
            nextDate = d2 - ql.Period(int(n*12/freq), ql.Months)
            prevDate = d2 - ql.Period(int(n*12/freq + 12/freq), ql.Months)
            if (t <= 1.0/freq):
                df = 1.0 + r/freq * (nextDate-d1) / (nextDate-prevDate)
            else:
                df = (1.0 + r/freq * (nextDate-d1) / (nextDate-prevDate)) * ((1 + r/freq)**n)
        else:
            df = super().compoundFactor(d1, d2)
        return df
