QuantLib: InterestRate Class ​

Language

  • English Page: 영어 페이지

InterestRate 클래스 개요 ​

금리 Interest Rate

  • FICC 는 Fixed Income, Currency, Commodity 를 말한다.
  • 여기서, Fixed Income 은 금리 관련 자산을 의미한다.
  • 금리란 일정 기간 동안 돈을 빌리거나 빌려줄 때 추가적으로 지불하거나 받아야 하는 대가를 의미한다.
  • 그런데 금리는 주가, 환율, 원자재 가격 등의 다른 기초자산들과는 다른 성질을 가지고 있다.
  • 그것은 바로 금리가 어떤 시점에 대한 개념이 아닌 기간에 대한 개념이라는 것이다.

InterestRate 클래스

  • 금리는 반드시 시점이 아닌 그 금리가 적용되는 특정 기간을 명시적으로 제시해야 한다.
  • 또한 금리가 가지고 있는 이러한 성질은 휴일 처리, 이자 일수 계산방식, 이자 복리계산방식 등에 대한 논의를 필요로 한다.
  • 퀀트립에서는 이러한 금리의 특이한 성질 때문에 금리를 다른 기초자산들과 구분지어 금리만을 위한 InterestRate 클래스를 만들어 놓았다.

InterestRate 클래스 구조 ​

python
import QuantLib as ql

# ql.InterestRate(rate: Real, 
#                 dayCount: ql.DayCounter,
#                 compounding: ql.Compounding,
#                 frequency: ql.Frequency)
rate = ql.InterestRate(0.05, ql.Actual360(), ql.Compounded, ql.Annual)

InterestRate 클래스

  • InterestRate 클래스는 주어진 시장 관행 방식들을 바탕으로 특정 기간 동안의 금리, 할인계수, 복리계수 등이 정확히 얼마인가를 계산하는 클래스이다.
  • 이 클래스를 정의하기 위해서는 아래와 같이 네 가지의 인자가 필요하다.
  • 각각 금리(rate), 이자 일수계산방식(DayCounter), 복리계산방식(Compounding), 이자지급주기(Frequency)이다.

Compounding

  • Compounding 은 복리계산방식을 어떻게 할 것인지를 결정하는 멤버들로 구성된 열거형이다.
  • 이 열거형의 하위 멤버들은 아래와 같이 다섯 가지이다.
  • 실제 금융의 세계에서는 기본적으로 대부분의 이자 계산이 일반적인 복리 방식을 취하고 있다.
  • 다만, 금융공학 모델링을 할 때에는 금리 모형을 연속복리로 가정한다.

Compounding

  • ql.Simple : 단리 [c = 1 + rt]
  • ql.Compounded : 복리 [c = (1 + r/f)^(tf)]
  • ql.Continuous : 연속복리 [c = e^(rt)]
  • ql.SimpleThenCompounded : 차기 이표일까지는 단리, 이후부터는 복리
  • ql.CompoundedThenSimple : 차기 이표일까지는 복리, 이후부터는 단리
    • c : 복리계수
    • r : 금리
    • t : 만기
    • f : 연간 이자지급횟수

관행적 복할인 방식

  • 실제 채권의 단가를 계산하는 방식은 관행적 복할인 방식으로, 위의 네 번째 ql.SimpleThenCompounded 방식과 유사하다.
  • 다만, 한국 내의 채권평가 관행은 퀀트립의 ql.SimpleThenCompounded 방식과 차이가 있다.
  • 차기 이표일까지의 복리계수 계산 시 실제 일수를 반영하기 때문이다.
  • 한국 채권평가 관행은 금융투자협회 > 금융투자회사의 영업 및 업무에 관한 규정 > [별표 14 : 채권수익률의 가격환산 산식] 에서 확인 가능하다.
  • 한국 채권평가 관행을 반영하여 수정한 InterestRate 클래스를 제일 아래에 첨부한다.

InterestRate 클래스 기능 ​

할인계수와 복리계수 계산 ​

  • discountFactor(d1, d2) 메서드 : 특정 기간 동안의 할인계수 계산

  • compoundFactor(d1, d2) 메서드 : 특정 기간 동안의 복리계수 계산

python
import QuantLib as ql

rate = ql.InterestRate(0.05, ql.Actual360(), ql.Compounded, ql.Annual)

# Discount Factor & Compound Factor
print("Rate: ", rate.rate())
print("DayCount: ", rate.dayCounter())
print("DiscountFactor: ", rate.discountFactor(1))
print("DiscountFactor: ", rate.discountFactor(ql.Date(15,6,2020), ql.Date(15,6,2021)))
print("CompoundFactor: ", rate.compoundFactor(ql.Date(15,6,2020), ql.Date(15,6,2021)))

# Output
# Rate:  0.05
# DayCount:  Actual/360 day counter
# DiscountFactor:  0.9523809523809523
# DiscountFactor:  0.9517357984551467
# CompoundFactor:  1.050711764360651

다른 시장 관행 하에서의 금리 계산 ​

  • equivalentRate(resultDC, comp, freq, d1, d2) : 다른 조건 하에서의 동일한 수준의 금리 반환

    • 이자 일수계산방식(resultDC), 복리계산방식(comp), 이자지급주기(freq), 시작일(d1), 종료일(d2)

  • impliedRate(compound, resultDC, comp, freq, d1, d2) : 어떤 복리계수가 주어지면 그 복리계수 안에 내재되어 있는 금리를 역산하여 반환

    • 복리계수(compound), 이자 일수계산방식(resultDC), 복리계산방식(comp), 이자지급주기(freq), 시작일(d1), 종료일(d2)
python
import QuantLib as ql

rate = ql.InterestRate(0.05, ql.Actual360(), ql.Compounded, ql.Annual)

print("Rate: ", rate.rate())
print("DayCount: ", rate.dayCounter())

print("EquivalentRate: ", rate.equivalentRate(ql.Actual360(), ql.Compounded, ql.Semiannual, ql.Date(15,6,2020), ql.Date(15,6,2021)))

factor = rate.compoundFactor(ql.Date(15,6,2020), ql.Date(15,6,2021))
print("ImpliedRate: ", rate.impliedRate(factor, ql.Actual360(), ql.Continuous, ql.Annual, ql.Date(15,6,2020), ql.Date(15,6,2021)))

# Output
# Rate:  0.05
# DayCount:  Actual/360 day counter
# EquivalentRate:  4.939015 % Actual/360 Semiannual compounding
# ImpliedRate:  4.879016 % Actual/360 continuous compounding

한국 관행적 복할인 반영: InterestRate 클래스 ​

한국 관행적 복할인 방식

  • 실제 한국 채권의 단가를 계산하는 방식은 관행적 복할인 방식으로, 위의 네 번째 ql.SimpleThenCompounded 방식과 유사하다.
  • 다만, 한국 내의 채권평가 관행은 퀀트립의 ql.SimpleThenCompounded 방식과 차이가 있다.
  • 차기 이표일까지의 복리계수 계산 시 실제 일수를 반영하기 때문이다.
  • 한국 채권평가 관행은 금융투자협회 > 금융투자회사의 영업 및 업무에 관한 규정 > [별표 14 : 채권수익률의 가격환산 산식] 에서 확인 가능하다.
  • 한국 채권평가 관행을 반영하여 수정한 InterestRate 클래스를 제일 아래에 첨부한다.

아래는 한국 채권평가 관행을 반영하여 수정한 InterestRate 클래스이다.

python
class InterestRate(ql.InterestRate):
    """Modify QuantLib InterestRate Class
       SimpleThenCompounded in QuantLib is not Korea Treasury Compounding Convention
       if compounding == ql.SimpleThenCompounded:
           for t <= 1/f : c = 1 + r*t
           for t > 1/f  : c = (1 + r/f)^(tf)
           where t = days / first_period_days
    """
    def __init__(self, r, dc, comp, freq):
        super().__init__(r, dc, comp, freq)
        self._r = r
        self._dc = dc
        self._comp = comp
        self._freq = freq.real
    
    def discountFactor(self, d1, d2):
        r = self._r
        dc = self._dc
        comp = self._comp
        freq = self._freq
        t = dc.yearFraction(d1, d2)
        if comp == ql.SimpleThenCompounded:
            n = int(np.divmod(t, 1/freq)[0])
            nextDate = d2 - ql.Period(int(n*12/freq), ql.Months)
            prevDate = d2 - ql.Period(int(n*12/freq + 12/freq), ql.Months)
            if (t <= 1.0/freq):
                df = 1 / (1.0 + r/freq * (nextDate-d1) / (nextDate-prevDate))
            else:
                df = 1 / ((1.0 + r/freq * (nextDate-d1) / (nextDate-prevDate)) * ((1 + r/freq)**n))
        else:
            df = super().discountFactor(d1, d2)
        return df
    
    def compoundFactor(self, d1, d2):
        r = self._r
        dc = self._dc
        comp = self._comp
        freq = self._freq
        t = dc.yearFraction(d1, d2)
        if comp == ql.SimpleThenCompounded:
            n = int(np.divmod(t, 1/freq)[0])
            nextDate = d2 - ql.Period(int(n*12/freq), ql.Months)
            prevDate = d2 - ql.Period(int(n*12/freq + 12/freq), ql.Months)
            if (t <= 1.0/freq):
                df = 1.0 + r/freq * (nextDate-d1) / (nextDate-prevDate)
            else:
                df = (1.0 + r/freq * (nextDate-d1) / (nextDate-prevDate)) * ((1 + r/freq)**n)
        else:
            df = super().compoundFactor(d1, d2)
        return df
