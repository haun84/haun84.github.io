테크니컬 SEO 개요 ​

  • 테크니컬 SEO는 웹사이트가 검색 엔진에 쉽게 인식될 수 있게 만드는 작업이다. 사이트맵, 로딩 속도, 모바일 호환성 등이 여기에 해당한다.

  • 웹사이트의 기술적인 구조는 웹사이트 퍼포먼스에 지대한 영향을 미치기 때문에 최적화 작업이 필수이다. 웹사이트가 아무리 많은 양질의 콘텐츠를 가지고 있다고 해도, 건강하고 탄탄한 기술적 구조를 갖추고 있지 않다면 해당 콘텐츠를 검색엔진은 물론 사용자에게도 효과적으로 전달할 수 없다.

  • 아래는 가장 기본적으로 적용해야 할 테크니컬 SEO 작업이다.

보안 프로토콜 (HTTPS) 적용하기 ​

  • 구글은 사용자가 구글을 통해 접속하는 모든 웹사이트가 안전할 수 있도록 보안 프로토콜을 랭킹 요소로 지정하여, 보안 프로토콜을 적용한 웹사이트(HTTPS)에게 보안 프로토콜을 적용하지 않은 웹사이트(HTTP)보다 더 높은 점수를 부여한다.

  • 모든 웹사이트에 보안 프로토콜(HTTPS) 적용을 강력히 권고하지만, 특히 웹사이트 상에서 금전적 거래, 사용자의 개인 정보를 수집하는 웹사이트의 경우는 필수적으로 적용해야 한다.

  • 보안 프로토콜을 적용하는 데는 여러 가지 방법이 있지만, 본 웹사이트처럼 깃허브(GitHub) 를 이용하여 웹사이트를 만드는 경우 무료로 HTTPS를 손쉽게 적용할 수 있고, 클라우드 플레어(CloudFlare) 등의 서비스를 이용하여 보안 프로토콜을 적용할 수 있다.

HaunBlog by GitHub Pages

  • HaunBlog는 GitHub의 웹 사이트 호스팅 서비스인 GitHub Pages를 이용하여 보안 프로토콜(HTTPS)를 적용 중이다.

모바일 친화성 갖추기 ​

  • 모바일 중심 색인 생성이란 모바일 버전의 콘텐츠를 우선적으로 색인한다는 의미이다. 모바일 검색량이 데스크톱 검색량을 추월하면서 구글은 본격적으로 모바일 중심 색인 생성을 시작하였다. 그렇다 보니 웹사이트의 모바일 친화성은 아주 중요한 랭킹 요소가 되었다.

  • 웹사이트의 모바일 친화성 여부는 구글의 모바일 친화성 테스트 도구를 통해 확인할 수 있으며, 만약 웹사이트가 모바일 버전을 지원하지 않는 경우, 반응형 웹디자인을 사용하여 웹사이트를 모바일 친화적으로 개선해야 한다.

  • 만약 모바일 전용 웹사이트 (예: m.example.com)을 사용 중인 경우, 모바일 웹사이트와 데스크톱 웹사이트의 상관관계를 알리는 캐노니컬 대그를 반드시 적용해야 한다.

페이지 로딩 속도 빠르게 하기 ​

  • 페이지 로딩 속도란 웹사이트 혹은 웹페이지를 클릭했을 때 로딩되기까지 시간이 얼마나 걸리는지를 의미한다.

  • 모바일 친화성과 마찬가지로 페이지 로딩 속도는 사용자 경험에 막대한 영향을 끼치지 때문에 보안 프로토콜, 모바일 친화성과 함께 구글이 가장 중요하게 생각하는 랭킹 요소 중 하나이다.

  • 구글은 데스크톱의 경우 3초 미만, 모바일의 경우 2초 미만의 로딩 속도를 보유한 웹사이트가 가장 경쟁력이 있다고 보고 있다. 웹사이트 로딩 속도는 구글이 제공하는 PageSpeed Insigths 도구를 통해 진단할 수 있다.

HaunBlog by VitePress

  • HaunBlog는 VitePress라는 빠르고 컨텐츠 중심의 웹사이트를 구축하기 위해 설계된 정적 사이트 생성기 (SSG)를 통해 작성하여 높은 모바일 친화성, 빠른 페이지 로딩 속도, 웹사이트 구조의 검색엔진 최적화를 적용 중이다.

URL을 구조적으로 생성하기 ​

  • 검색 엔진이 사이트 구조를 파악할 때 URL의 요소도 검토한다. 모든 게시글이 메인 도메인으로만 연결되었다면 아래와 같은 구성이 된다.

https://www.example.com/SEO-Guide
https://www.example.com/Service-Introduction
https://www.example.com/SEO-Marketing-Guide

  • 위의 경우 내용은 각각 다르지만 모두 https://www.example.com 으로만 묶이게 된다. 하지만 URL을 구조적으로 사용한다면 카테고리를 묶어서 SEO에 더 친화적인 구조를 만들 수 있다.

https://www.example.com/guide/SEO-Guide
https://www.example.com/service/Service-Introduction
https://www.example.com/marketing/SEO-Marketing-Guide

  • 위의 경우는 guide, service, marketing 폴더로 구분이 되고, 크롤링 봇이 웹사이트의 구조를 파악하기에 도움이 된다.

  • 또한 URL을 내용의 주제와 관련된 단어로 설정하는 것이 SEO 친화적이다.

사이트맵 (sitemap.xml) 생성 및 주기적으로 업데이트 하기 ​

  • 사이트맵이란 웹사이트에서 발행한 모든 웹 페이지를 긴 리스트로 정리한 지도라고 생각하면 된다.

  • 사이트맵은 검색엔진이 웹사이트를 방문하여 크롤링을 하는 과정에서 모든 웹페이지를 빠짐없이 색인해갈 수 있도록 도와준다.

  • 이에 웹사이트에 새로운 콘텐츠가 발행될 때마다 해당 URL을 사이트맵에 추가하여 주기적으로 사이트맵을 업데이트하는 것이 필요하다.

  • 구글 SEO 를 위해서 사이트맵을 구글 서치 콘솔에 직접 업데이트 등록 가능하다.

  • 사이트맵을 제작하고 제출하는 방법을 더욱 자세히 알고 싶다면 구글 가이드라인을 참고하면 된다.

HaunBlog sitemap

  • HaunBlog의 사이트맵은 https://haun84.github.io/sitemap.xml 에서 확인 가능하다.

로봇 배제 표준 파일 (robots.txt) 생성하기 ​

  • 로봇 배제 표준 파일이란 검색엔진의 크롤링을 하는 크롤러(로봇)의 접근을 차단하거나 허용하는 파일을 의미한다.

  • 이때 검색엔진은 구글뿐만이 아니라 네이버, 얀덱스, 빙, 다음, 야후 등 모든 검색엔진이 해당되며 모든 검색엔진 로봇에게 공통된 명령을 내릴 수 있고, 각 검색엔진 로봇별로 다른 명령을 내릴 수도 있다.

모두 허용

  • 예를 들어, 웹사이트의 모든 페이지를 모든 검색엔진 크롤러에게 색인 허용하고 싶다면 아래와 같이 로봇 배제 표준 파일을 작성하면 된다.
txt
User-agent: *
Allow: /

일부 금지

  • 반대로 모든 검색엔진 크롤러(로봇)으로부터 웹사이트의 관리자 페이지 (예: www.example.com/admin/)를 검색 결과 페이지에 노출 되지 않도록 색인 불허 명령을 내릴려면 아래와 같이 로봇 배제 표준 파일을 작성하면 된다.
txt
User-agent: *
Disallow: /admin/

모두 금지

  • 주의할 것은 아래와 같이 작성할 경우 모든 크롤러에게 모든 콘텐츠 색인을 금지하는 것으로 조심해야 한다.
txt
User-agent: *
Disallow: /
